# 移行メモ

[English migration notes](../MIGRATION.md)

## ローカルアーカイブログ

ThreadForge はローカルアーカイブ形式の `LOG_*.cgi` ファイルをインポートできます。ただし、この操作は意図的にWeb管理画面には置いていません。`server/db.php` の `importLocalArchiveDirectory()` を呼び出すローカル運用バッチ、またはローカルPHPコマンドから実行します。

既定の読み込み元:

- `data/`

想定レイアウト:

```text
data/
  LOG_009359.cgi
  LOG_009360.cgi
  DOTIMG_009359_1.gif
  DOTIMG_009360.png
```

インポート動作:

- 各 `LOG_*.cgi` の先頭行は親スレッドになります。
- 2行目以降はそのスレッドの返信になります。
- 参照画像は `server/storage/data/` へコピーします。
- 既存DB行、アップロード画像、管理設定はリセットしません。
- 再実行時は、名前、本文、時刻が一致する投稿と返信をスキップします。

インポートされた投稿は未知の生成済みパスワードハッシュを使います。管理は管理画面から行います。

個別環境用のローカルバッチは、プロジェクトとして配布すると決めない限りコミットしません。

## フルバックアップ復元

バックアップJSONのインポートはローカルアーカイブインポートとは異なります。これは完全復元であり、投稿と画像をバックアップ内容で置き換えます。

ThreadForge インスタンスを別環境へ移す場合はバックアップ復元を使います。既存掲示板へ過去ログを追加する場合はローカルアーカイブインポートを使います。
