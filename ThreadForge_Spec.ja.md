# ThreadForge 現在仕様

[English specification](ThreadForge_Spec.md)

ThreadForge は、投稿、画像、返信、モデレーション、コミュニティアーカイブを扱う軽量でカスタマイズ可能なスレッド掲示板エンジンです。

ThreadForge は、作品投稿、メディア投稿、制作ログ、小規模コミュニティ、アーカイブ用途に適用できる新しいセルフホスト型掲示板エンジンです。

## ランタイム構成

- `client/`: React 18 + TypeScript + Vite の SPA
- `server/api.php`: PHP API エントリポイント
- `server/db.php`: SQLite 接続、スキーマ初期化、ストレージ補助
- `server/database.sqlite`: 実行時 SQLite DB。Git 管理外
- `server/storage/data/`: アップロード画像。`.gitkeep` 以外は Git 管理外
- `docs/`: プロジェクトドキュメント

## ルート

- `/`: スレッド一覧
- `/thread/:id`: スレッド詳細、返信、返信フォーム
- `/thread/:id?mode=eejanaika`: 固定文返信フォーム
- `/post`: 新規親投稿フォーム
- `/search`: 検索ページ
- `/delete`: 削除モード
- `/edit`: 編集モード
- `/edit/:id`: 指定投稿または返信の編集
- `/admin`: 管理画面
- `/manual`: 組み込みマニュアルページ

## コア機能

- 親投稿の作成
- 返信の作成
- 親投稿のみ画像アップロード可能
- 返信では画像アップロード不可
- 親投稿のみSNS転記プレビューとSNS転記
- X、Bluesky、Mastodon、Misskey 連携はデフォルトOFF
- 親投稿のSNS転記先URL/ID保存とリアクションキャッシュ表示
- SNS転記OFFモード
- gdgd 投稿モード
- 表示番号は親投稿だけに割り当て、返信はスレッド内の子番号を使う
- クラシックなトップナビゲーション
- トップページにスレッドごとの返信を最大10件表示
- コメントまたは「ええじゃないか」送信後、スレッド詳細ページに留まる
- 内部データを残すソフトデリート
- 投稿パスワードによる編集と削除
- 管理画面からの削除済み投稿復元
- 投稿パスワード不要の管理者一括削除
- 管理画面のDB整合性チェック
- DB、画像、設定を含む管理画面バックアップのエクスポート/インポート
- ローカル運用バッチまたはPHPコマンドによるローカルアーカイブ `LOG_*.cgi` の非破壊インポート
- `config.cgi` と `skincfg.cgi` 相当の管理設定保存
- 管理画面からの HOME リンク設定
- 管理画面からの組み込みマニュアルタイトル/本文設定
- 全文、タイトル、本文、作者での検索
- RSSフィード

## 管理機能

管理画面はバックエンドの `DOTEITA_ADMIN_PASSWORD` を使います。

実装済み:

- 通常投稿と削除済み投稿の読み込み
- 個別投稿パスワードなしの投稿/返信一括削除
- 削除済み投稿の復元
- DB整合性チェック
- DB行、画像、設定を含むバックアップJSONのエクスポート
- バックアップJSONのインポートによるDB/画像/設定復元
- 既存DB、画像、設定をリセットしないローカルアーカイブログのローカルインポート
- 現在のアプリ設定の編集と保存
- トップナビゲーション HOME リンクの設定
- SPA組み込みマニュアルのタイトル/本文設定
- SNS連携のON/OFF
- gdgd投稿UIのON/OFF
- gdgd投稿ラベルの設定
- 管理者パスワード変更
- SNSリアクション数キャッシュの更新
- SNSリアクション自動更新用のローカルCronパスとAPIキー付きURLの表示

メモ:


- 代替の保守機能では、DB整合性と画像参照の欠落を確認します。
- 保存済み `config` と `skin` 設定は管理APIで永続化します。
- `homePageUrl`、`manualTitle`、`manualBody`、SNSスイッチ、`gdgdEnabled`、`gdgdLabel`、基本色は公開設定APIを通じてUIへ反映されます。
- 残りの保存済み設定をすべてランタイム挙動とライブスタイルへ反映する作業は進行中です。

## データモデル

主テーブルは SQLite の `posts` です。

主なカラム:

- `id`
- `thread_id`
- `parent_id`
- `name`
- `url`
- `title`
- `message`
- `image_path`
- `password_hash`
- `created_at`
- `deleted_at`
- `gdgd`
- `tweet_off`
- `tweet_text`
- `tweet_url`
- X/Tweet のいいね、リポスト、閲覧数カラム
- Bluesky の URI/CID/URL と Like/Repost/Quote 数
- Mastodon の ID/URL と Boost/Fav 数
- Misskey の ID/URL とリアクション数

`settings` テーブルは、管理画面で扱う設定セクションをJSONとして保存します。`security` セクションには `adminPasswordHash` と `cronApiKey` を含みます。

公開UIは `?action=publicSettings` から安全な設定だけを読みます。

## SNS転記ルール

- SNS転記は、投稿/編集フォームの `SNS転記OFF` スイッチで制御します。
- 管理画面では X、Bluesky、Mastodon、Misskey の設定を分けます。
- 各SNSがOFFの間、そのSNSの認証情報欄は無効化されます。
- 新規の親投稿だけが、有効なSNSへ転記されます。
- 親投稿の編集は掲示板内だけに反映し、SNS側の編集・再投稿は行いません。
- 返信はSNS転記しません。返信フォームにはSNSコントロールを表示しません。
- 画像付き親投稿では、X、Bluesky、Mastodon、Misskey へ画像も送信します。
- 有効なSNSごとに読み取り専用のプレビューを表示します。
- `_TWEND_` 以降の本文はSNS転記対象外です。
- SNS本文には `最新はこちら` と `/#post-000001` のような掲示板一覧リンクを含めます。一覧ページでは該当投稿位置へスクロールします。
- 文字数上限を超える場合、ThreadForge は `..` で省略して送信します。
- 既定上限は X 280、Bluesky 300、Mastodon 500、Misskey 3000 です。
- X は重み付き文字数、それ以外は通常の文字数でプレビューします。
- キャッシュ済みリアクション数は同色のシンプルな指標として表示します。
- リアクション更新は管理画面、ローカル `server/cron.php`、または `?action=cronRefreshSocialReactions&api_key=...` から実行できます。
- 自動リアクション更新は、作成から7日以内の未削除親投稿だけを対象にします。

## コメントと固定文返信フォーム

- コメントフォームは名前、URL/HOME、本文、パスワードを使います。
- コメントでは名前、本文、パスワードが必須です。
- UI上の案内では、パスワードは8文字以内の半角英数字です。
- 固定文返信フォームは名前を入力し、固定返信文を1つ選びます。
- インラインコメントと固定文返信フォームには閉じるボタンがあり、下書きを閉じるときは確認します。
- インラインフォームを開いている間、スレッド操作ボタンは隠します。

## ストレージルール

- アップロード画像は `server/storage/data/` に保存します。
- 実行時DBとアップロード画像は Git 管理外です。
- バックアップは管理画面のエクスポート機能から行います。
- 投稿画像を差し替える場合、古い画像はすぐ削除せず履歴名へ退避します。
- ローカルアーカイブログインポートは画像をストレージへコピーし、名前・本文・時刻が一致する既存投稿/返信をスキップします。Web管理画面ではなくローカルで実行します。
- クリーン初期化はサーバー停止後、`server/database.sqlite` と `server/storage/data/` 配下のファイルを削除して行います。詳細は `docs/ja/DB.md` を参照してください。

## 削除ルール

削除はソフトデリートです。

- 通常UIから投稿が見えなくなります。
- DBには `deleted_at` 付きでデータが残ります。
- 管理画面から復元できます。
- 親スレッドを削除すると、その返信もソフトデリートします。

## 公開名称

公開プロジェクト名は `ThreadForge` です。

公開UI、パッケージメタデータ、リポジトリメタデータ、公開ドキュメントでは `ThreadForge` を新規ソフトとして扱います。

## バージョン管理

プロジェクトはセマンティックバージョニングで管理します。

現在のバージョン: `0.1.0`

バージョン参照は次のファイルで同時に更新します。

- `VERSION`
- `CHANGELOG.md`
- `client/package.json`
- `client/src/version.ts`

PHP API は `?action=version` で現在バージョンを返します。
